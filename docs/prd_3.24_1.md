# OpenMindLearn 功能缺口记录（2026-03-24）

文档来源：`docs/PROJECT_REQUIREMENTS.md` 对照当前代码实现（不含部署相关，如 Docker/客户端）。

## 结论概览

- 已实现主链路：节点创建与拖拽、划线扩展（直接展开/针对性提问/自定义上下文）、上下文 XML 传递、`.oml` 整图保存与加载、LLM 基础配置（API Key/Base URL/Model）。
- 2026-03-24 本轮已补完：初始节点创建入口、Markdown GFM、`resources/images`+`resources/attachments` 子目录、多父链手动上下文选择。
- 当前主要缺口集中在：版本控制、标签备注区域、搜索、单节点导出、LLM 高级参数、上下文深度可配置。

## 已实现（可用）

1. 节点卡片与画布交互（React Flow 拖拽、连线、右键创建）。
2. 划线后扩展流程（直接展开、针对性提问、自定义上下文展开）。
3. 上下文链式传递（默认自动回溯 + 手动勾选）。
4. 上下文 XML 构建并传给 LLM。
5. `.oml`（ZIP）保存/加载：`structure.xml + nodes/*.md + resources/`。
6. 来源片段定位（`sourceRef`）与来源节点着色恢复机制。
7. LLM 配置弹窗（`apiKey/baseURL/model`）与后端运行时配置更新。

## 本轮已补完（2026-03-24）

1. 初始节点创建入口
- 结果：空画布新增首节点创建面板，支持“粘贴文本创建首节点 / 输入 prompt 直接生成首节点”。

2. Markdown 富文本支持
- 结果：已接入 `remark-gfm`，节点卡片与详情面板支持 GFM（表格/任务列表/删除线等）。

3. 资源目录结构
- 结果：`.oml` 保存时创建 `resources/`、`resources/images/`、`resources/attachments/`。

4. 手动上下文选择能力
- 结果：上下文面板改为遍历全部上游父链，支持跨分支混选，并按前端勾选顺序传给后端生成上下文 XML。

## 未实现（明确缺失）

1. 节点版本控制
- 缺：每节点最多 3 版本、历史查看、Diff 视图、恢复版本。

2. 节点标注与组织
- 缺：标签（多标签）、备注、区域标注（分组框、颜色、标题编辑）。

3. 搜索
- 缺：按内容/标签/备注搜索与结果高亮。

4. 单节点导出
- 缺：导出单个节点为 `.md`。

5. LLM 高级参数
- 缺：`temperature`、`max_tokens` 等参数 UI、持久化与后端透传。

6. 上下文回溯深度可配置
- 现状：后端写死 `10`。
- 缺：设置项与请求透传。

## 建议开发优先级（供后续排期）

### P0（先做）
1. 上下文深度可配置（设置 -> 请求 -> 后端生效）。
2. 单节点导出 `.md`。
3. LLM 参数补齐（temperature/max_tokens）。

### P1
1. 搜索（先内容，再扩展到标签/备注）。
2. 标签/备注系统（数据结构 + UI）。

### P2
1. 区域标注（分组框）。
2. 节点版本控制（3 版本 + Diff + 恢复）。
3. `.oml` 资源子目录规范化与附件流转。
