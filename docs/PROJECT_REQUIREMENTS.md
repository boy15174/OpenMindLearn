# OpenMindLearn 项目需求文档

## 项目概述

OpenMindLearn 是一个基于网状知识图谱的交互式学习系统。用户可以从任意知识点开始，通过划线提问的方式逐层深入探索，系统利用 LLM 生成扩展内容，形成个性化的知识网络。

## 核心理念

- **网状结构**：没有严格的层级限制，知识节点可以任意连接
- **按需扩展**：通过划线提问的方式，只在需要时深入
- **上下文感知**：支持携带学习路径上下文，生成更准确的内容
- **本地优先**：知识图谱保存在本地，支持导入导出

---

## 功能需求

### 1. 知识节点创建与展示

**描述**：系统以卡片式布局展示知识节点，支持自由拖拽和网状结构。

**核心要素**：
- ✅ **卡片式布局**：每个知识节点以卡片形式展示，可以在画布上自由拖拽定位
- ✅ **初始节点创建方式**：
  - 用户直接粘贴文本内容作为第一个节点
  - 用户输入 prompt，点击"基于当前输入生成"按钮，由 LLM 生成节点内容
- ✅ **网状结构**：不限制层级，没有严格的"根节点"概念，节点之间可以形成任意连接关系
- ✅ **节点持久化**：每个知识节点保存为独立的本地文件

---

### 2. 文本划线与知识扩展

**描述**：用户可以在任何卡片内容中划线选中文本，通过弹框选择扩展方式来生成新的知识节点。

**核心要素**：
- ✅ **文本划线**：用户可以在卡片内容中选中任意文本片段
- ✅ **扩展弹框**：划线后弹出操作框，包含两个按钮：
  - **"直接展开"**：基于选中文本和系统 prompt 直接生成新节点
  - **"针对性提问"**：允许用户修改最终的 prompt（可以完全改写）
- ✅ **LLM 生成**：由配置的 LLM 引擎实时生成扩展内容
- ✅ **新节点创建**：生成的内容作为新的知识节点卡片，与原节点建立连接关系

---

### 3. 上下文链式传递

**描述**：系统记录节点之间的链式关系，在生成新节点时可以携带上游上下文，实现类似 chatbot 的连续对话效果。

**核心要素**：
- ✅ **链式关系记录**：使用链表结构记录节点的父子关系（A → B → C → D）
- ✅ **上下文传递**：
  - 当从节点 C 扩展出新节点 D2 时，可以将 A + B + C 的内容作为上下文传递给 LLM
  - 上下文包含节点内容和用户的提问部分
  - 默认回溯深度为 10 个节点，可在设置中修改
- ✅ **上下文格式化**：采用 XML 格式，便于处理复杂的回复结果
- ✅ **分支支持**：同一节点可以扩展出多个分支（如 C → D 和 C → D2）
- ✅ **手动选择上下文**：可打开上下文选择面板
  - 滚动列表形式，每行 item 显示节点摘要和勾选状态
  - 用户可以勾选/取消不同消息流中的节点

---

### 4. 知识图谱文件格式

**描述**：整个知识图谱以专有格式保存，便于导入导出和分享。

**核心要素**：
- ✅ **文件格式**：`.oml` 格式（OpenMindLearn 专有格式）
- ✅ **本质结构**：ZIP 压缩包，包含：
  - **结构文件**：XML 格式，记录节点关系、位置、连接等元数据
  - **节点内容文件**：每个节点的内容保存为独立文件
  - **资源文件夹**：存储图片、附件等资源
- ✅ **优势**：
  - 便于版本控制和备份
  - 支持导入导出和分享
  - 可以解压查看和手动编辑

**文件结构示例**：
```
knowledge_graph.oml (实际是 ZIP)
├── structure.xml          # 图谱结构和元数据
├── nodes/                 # 节点内容目录
│   ├── node_001.md
│   ├── node_002.md
│   └── node_003.md
└── resources/             # 资源文件目录
    ├── images/
    └── attachments/
```

---

### 5. LLM 引擎配置

**描述**：系统应该支持配置不同的 LLM 引擎来生成知识内容。

**核心要素**：
- ✅ **初期实现**：支持单一模型直接调用（如 Gemini）
- ✅ **配置界面**：提供 UI 让用户设置：
  - Base URL（支持自定义端点）
  - API Key
  - Model 名称
  - 基础参数（temperature、max_tokens 等）
- ✅ **未来扩展**：使用 LangGraph.js 实现 Agent 架构

---

### 6. 部署与分发

**描述**：支持多种部署方式，适配不同的使用场景。

**核心要素**：
- ✅ **Docker 部署**：
  - 提供 Docker Compose 配置，一键拉起
  - 服务组件：前端（Web UI）+ 后端 API
  - 通过环境变量配置 LLM API Key
- ✅ **本地桌面应用**（优先支持 macOS）：
  - 使用 Electron / Tauri 打包为原生 App
  - 支持 macOS DMG 安装
  - 未来可扩展到 Windows / Linux
- ✅ **数据存储**：无需数据库，使用本地文件存储（类似 Excel，全部加载到内存）
- ✅ **单用户模式**：本地应用，不支持多用户

---

### 7. 节点版本控制

**描述**：每个节点支持保存最多 3 个版本的修改历史，便于回溯和对比。

**核心要素**：
- ✅ **版本数量**：单节点最多保存 3 个历史版本
- ✅ **版本粒度**：以单个节点为单位进行版本管理
- ✅ **版本操作**：
  - 查看历史版本
  - 展示版本差异（Diff 对比视图）
  - 恢复到某个历史版本

---

### 8. 节点标注与组织

**描述**：支持对节点进行标签、备注和区域标注，便于组织和管理知识图谱。

**核心要素**：
- ✅ **标签系统**：可以给节点添加多个标签（如"重要"、"待复习"、"已掌握"）
- ✅ **备注功能**：可以给节点添加个人备注
- ✅ **区域标注**：类似 UE 蓝图的区域框
  - 可以框选多个节点并添加区域说明
  - 支持选择区域颜色
  - 左上角显示区域名称框，双击可编辑
- ✅ **搜索功能**：支持按关键词搜索，范围包括节点内容、标签、备注

---

### 9. 内容渲染与导出

**描述**：节点内容以 Markdown 格式存储和渲染，支持导出单个节点。

**核心要素**：
- ✅ **Markdown 渲染**：节点内容默认以 Markdown 格式渲染显示
- ✅ **单节点导出**：支持将单个节点导出为 `.md` 文件
- ✅ **富文本支持**：支持代码块、表格、图片等 Markdown 元素

## 用户交互流程

### 流程 1：创建初始节点
1. 用户打开应用，看到空白画布
2. 用户可以选择：
   - 粘贴文本内容，直接创建节点
   - 输入 prompt（如"Kubernetes 基础知识"），点击"生成"按钮
3. 系统创建第一个知识节点卡片

### 流程 2：扩展知识节点
1. 用户在某个节点卡片中阅读内容
2. 遇到不理解的概念，用鼠标划线选中
3. 弹出操作框，显示两个按钮
4. 用户选择：
   - **直接展开**：系统立即调用 LLM 生成解释
   - **针对性提问**：弹出输入框，用户可以修改 prompt
5. 系统生成新的知识节点，并在画布上显示连接线

### 流程 3：深度探索
1. 用户在新生成的节点中继续划线提问
2. 系统根据配置决定是否携带上下文（A → B → C）
3. 生成新的分支节点
4. 用户可以在画布上自由拖拽节点，调整布局

### 流程 4：保存与分享
1. 用户点击"保存"按钮
2. 系统将整个知识图谱打包为 `.oml` 文件
3. 用户可以将文件分享给他人
4. 其他用户可以导入该文件，继续扩展

## 技术架构建议

### 前端
- **框架**：React + TypeScript
- **画布库**：React Flow（用于节点拖拽、连接线、区域标注）
- **Markdown 渲染**：react-markdown / marked
- **UI 组件**：Ant Design / shadcn/ui
- **状态管理**：Zustand（轻量级，适合单用户应用）

### 后端
- **语言**：Node.js + TypeScript（与前端技术栈统一）
- **框架**：Express / Fastify
- **LLM 集成**：
  - 通用 HTTP 客户端（支持自定义 Base URL）
  - Google Generative AI SDK（Gemini）
  - OpenAI SDK
  - Anthropic SDK（Claude）
- **Agent 框架**：LangGraph.js
- **文件处理**：
  - JSZip（ZIP 压缩/解压）
  - xml2js（XML 解析）
  - fs-extra（文件操作）

### 数据存储
- **方案**：纯文件系统 + 内存缓存
- **结构**：
  - `.oml` 文件（ZIP 格式）
  - 运行时全部加载到内存（类似 Excel）
  - 修改后保存回文件

### 部署与分发
- **Docker**：前端（Nginx）+ 后端（Node.js），Docker Compose 一键启动
- **桌面应用**：Electron / Tauri 打包，优先支持 macOS
- **Volume 挂载**：持久化用户数据和配置
